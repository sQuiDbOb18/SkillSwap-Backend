# SkillSwap API Reference

This is the route/notes document to copy into an external Swagger/OpenAPI editor for the mobile developer. It is not generated by the backend.

Base URL in development: `http://localhost:4000`

API base path: `/api/v1`

Auth header for protected routes:

```http
Authorization: Bearer <accessToken>
```

Common JSON error shape:

```json
{
  "success": false,
  "type": "VALIDATION_ERROR",
  "message": "Validation failed",
  "errors": [{ "field": "email", "message": "Invalid email address" }]
}
```

## Health

`GET /health`

Checks API and database connection.

`GET /health/ready`

Readiness check. Returns `204` when the database is reachable.

## Auth

`POST /auth/register`

Starts signup and emails a 4-digit verification code.

Body:

```json
{
  "name": "Ada Lovelace",
  "email": "ada@example.com",
  "password": "Password@123",
  "confirmPassword": "Password@123"
}
```

`POST /auth/verify-email`

Completes signup from pending registration.

Body:

```json
{ "code": "1234" }
```

`POST /auth/resend-verification`

Resends signup verification code for a pending registration.

Body:

```json
{ "email": "ada@example.com" }
```

`POST /auth/login`

Logs in a verified user. Also sets the refresh token cookie.

Body:

```json
{
  "email": "ada@example.com",
  "password": "Password@123"
}
```

Returns:

```json
{
  "accessToken": "jwt",
  "refreshToken": "jwt"
}
```

`POST /auth/refresh`

Refreshes tokens. Uses refresh token cookie or refresh token request support in the backend helper.

`POST /auth/logout`

Protected. Clears refresh token server-side and invalidates current token version.

## Users

`GET /user/verify-email`

Email-change redirect endpoint. Query uses `token`. Browser-facing route.

`GET /user/profile`

Protected. Gets the authenticated user's profile, skills, and session stats.

`PUT /user/profile`

Protected. Updates profile fields.

Body fields:

```json
{
  "fullName": "Ada Lovelace",
  "username": "ada_lovelace",
  "age": 22,
  "phoneNumber": "08012345678",
  "bio": "Backend learner",
  "profileImage": "https://example.com/avatar.png"
}
```

`POST /user/profile-image`

Protected. Uploads a profile image to Cloudinary and saves the secure URL to `User.profileImage`.

Content type: `multipart/form-data`

Field:

```txt
image
```

Allowed formats: JPG, PNG, WEBP. Max size: 5MB.

`PUT /user/password`

Protected. Changes password and invalidates refresh sessions.

Body:

```json
{
  "currentPassword": "OldPassword@123",
  "newPassword": "NewPassword@123",
  "confirmNewPassword": "NewPassword@123"
}
```

`PUT /user/email`

Protected. Starts email-change flow and sends a verification code to the new email.

Body:

```json
{ "email": "new@example.com" }
```

`POST /user/verify-email`

Verifies email-change code.

Body:

```json
{ "code": "1234" }
```

`POST /user/forgot-password`

Sends a 6-digit reset code.

Body:

```json
{ "email": "ada@example.com" }
```

`POST /user/reset-password/verify-code`

Checks if reset code is valid.

Body:

```json
{ "code": "123456" }
```

`POST /user/reset-password`

Completes password reset.

Body:

```json
{
  "code": "123456",
  "newPassword": "NewPassword@123",
  "confirmPassword": "NewPassword@123"
}
```

`POST /user/restore-account`

Restores a soft-deleted account and returns fresh tokens.

Body:

```json
{
  "email": "ada@example.com",
  "password": "Password@123"
}
```

`DELETE /user/account`

Protected. Soft deletes authenticated account.

Body:

```json
{
  "password": "Password@123",
  "reason": "Taking a break"
}
```

## Skills

`GET /skills/discover`

Public. If a valid bearer token is provided, the search is saved to the user's search history.

Query params:

- `search`
- `category`: `FRONTEND`, `BACKEND`, `FULLSTACK`, `MOBILE`, `DEVOPS`, `DATA_SCIENCE`, `AI_ML`, `UI_UX`, `CYBERSECURITY`, `CLOUD`, `QA`, `OTHER_TECH`
- `minRating`: `0` to `5`
- `availabilityDay`: `MONDAY` to `SUNDAY`
- `level`: `BEGINNER`, `INTERMEDIATE`, `ADVANCED`, `EXPERT`
- `tag`
- `isBarter`: `true` or `false`
- `minCreditCost`
- `maxCreditCost`
- `sortBy`: `newest`, `rating`, `creditCostAsc`, `creditCostDesc`
- `page`
- `limit`: max `50`

`GET /skills`

Protected. Lists authenticated user's own skills.

`GET /skills/recommendation`

Protected. Returns a skill recommendation based on existing user skills.

`GET /skills/favorites`

Protected. Lists favorited public skills.

`GET /skills/search-history`

Protected. Lists recent authenticated discovery searches.

`GET /skills/saved-searches`

Protected. Lists saved searches.

`POST /skills/saved-searches`

Protected. Creates or updates a saved search by name.

Body:

```json
{
  "name": "React swaps",
  "query": {
    "search": "react",
    "category": "FRONTEND",
    "isBarter": true,
    "page": 1,
    "limit": 10
  }
}
```

`DELETE /skills/saved-searches/:savedSearchId`

Protected. Deletes one saved search owned by the user.

`POST /skills/:skillId/favorite`

Protected. Favorites a public skill. Users cannot favorite their own skill.

`DELETE /skills/:skillId/favorite`

Protected. Removes a favorite.

`GET /skills/:skillId`

Protected. Gets one skill owned by the authenticated user.

`POST /skills`

Protected. Creates a skill.

Body:

```json
{
  "title": "Build REST APIs with Express",
  "description": "Learn Express routing, validation, and services.",
  "category": "BACKEND",
  "level": "INTERMEDIATE",
  "tags": ["express", "typescript"],
  "availability": [
    { "day": "MONDAY", "startTime": "09:00", "endTime": "10:00" }
  ],
  "price": null,
  "creditCost": 20,
  "isBarter": false
}
```

Note: `creditCost` is required for non-barter skills.

`PUT /skills/:skillId`

Protected. Updates an owned skill. Same fields as create, all optional, but at least one field is required.

`DELETE /skills/:skillId`

Protected. Deletes an owned skill.

## Bookings

Booking rules:

- Protected routes.
- User cannot book their own skill.
- New booking checks overlapping pending/confirmed sessions for requester and provider.
- Pending bookings hide `meetingLink` and `meetingPlatform` in responses.
- Confirmed/completed bookings show meeting details.
- `LEARN` bookings can hold requester credits.
- `SWAP` bookings require `offeredSkillId` and both skills must have positive `creditCost`.

`GET /bookings`

Lists bookings where the authenticated user is requester or skill provider.

`POST /bookings`

Creates a booking request.

Body for paid/learn session:

```json
{
  "skillId": "skill-id",
  "type": "LEARN",
  "date": "2026-06-01T10:00:00.000Z",
  "sessionMode": "EXTERNAL_LINK",
  "meetingPlatform": "ZOOM",
  "meetingLink": "https://zoom.example/session",
  "durationMinutes": 60,
  "sessionNotes": "I want to focus on API auth."
}
```

Body for swap:

```json
{
  "skillId": "requested-skill-id",
  "type": "SWAP",
  "offeredSkillId": "my-offered-skill-id",
  "date": "2026-06-01T10:00:00.000Z",
  "sessionMode": "EXTERNAL_LINK",
  "meetingPlatform": "GOOGLE_MEET",
  "meetingLink": "https://meet.google.com/abc-defg-hij",
  "durationMinutes": 60
}
```

Validation notes:

- `date` must be future ISO datetime with timezone offset.
- `durationMinutes`: 15 to 240.
- `sessionMode`: `EXTERNAL_LINK` or `IN_PERSON`.
- If `IN_PERSON`, do not send `meetingLink` or `meetingPlatform`.
- If external link is accepted, meeting link/platform must exist.

`PATCH /bookings/:bookingId/respond`

Provider accepts or rejects a pending booking.

Accept body:

```json
{
  "action": "accept",
  "meetingPlatform": "GOOGLE_MEET",
  "meetingLink": "https://meet.google.com/abc-defg-hij",
  "sessionNotes": "Bring your laptop."
}
```

Reject body:

```json
{ "action": "reject" }
```

Notes:

- Only skill owner/provider can respond.
- Only pending bookings can be accepted/rejected.
- For external sessions, accept requires meeting link and platform.
- Reject refunds held credits.

`PATCH /bookings/:bookingId/cancel`

Requester or provider cancels a pending/confirmed booking.

Body:

```json
{ "reason": "Schedule changed" }
```

Notes:

- Refunds held credits.
- Marks outcome as `CANCELLED_BY_REQUESTER` or `CANCELLED_BY_PROVIDER`.

`PATCH /bookings/:bookingId/extension`

Requests additional time for a confirmed booking.

Body:

```json
{
  "extraMinutes": 30,
  "extraCreditCost": 10
}
```

Notes:

- `extraMinutes`: 15 to 180.
- For `LEARN`, only requester can request extension and `extraCreditCost` must be greater than zero.
- For `SWAP`, either participant can request; `extraCreditCost` must be a positive even number so it can split evenly.
- Only one pending extension at a time.

`PATCH /bookings/:bookingId/extension/respond`

Approves or rejects a pending extension request.

Body:

```json
{ "action": "approve" }
```

or

```json
{ "action": "reject" }
```

Notes:

- Only the other participant can respond.
- Approve holds additional credits where required.

`PATCH /bookings/:bookingId/complete`

Provider marks a confirmed booking completed.

Notes:

- Only provider can complete.
- Releases held credits as payouts.

`PATCH /bookings/:bookingId/outcome`

Reports a no-show outcome for a confirmed booking.

Body:

```json
{ "outcome": "REQUESTER_NO_SHOW" }
```

Allowed outcomes:

- `REQUESTER_NO_SHOW`
- `PROVIDER_NO_SHOW`
- `BOTH_NO_SHOW`

Notes:

- Only participants can report.
- A participant cannot report themself as the no-show.
- Handles refunds/partial compensation depending on booking type.

## Messages

`GET /messages/conversations`

Protected. Lists conversations with last message, unread count, presence, and last seen.

`GET /messages/unread-count`

Protected. Returns unread message count.

`POST /messages`

Protected. Sends a direct message.

Body:

```json
{
  "receiverId": "user-id",
  "message": "Hey, are you available tomorrow?"
}
```

`PATCH /messages/:userId/read`

Protected. Marks conversation with user as read.

`GET /messages/:userId`

Protected. Gets chat history with one user and marks received messages as read.

Socket events:

- Client emits `chat:send` with `{ receiverId, message }`.
- Client emits `chat:read` with `{ userId }`.
- Client emits `chat:typing` with `{ receiverId, isTyping }`.
- Server emits `chat:message`, `chat:delivered`, `chat:read`, `chat:typing`, `presence:update`.

Typing event payload from server:

```json
{
  "userId": "typing-user-id",
  "senderId": "typing-user-id",
  "receiverId": "other-user-id",
  "isTyping": true
}
```

Mobile note: use the same `chat:typing` event for both the open chat screen and the conversation list. When `isTyping` is `true`, show "typing..." for that conversation. When `false`, hide it. The server also sends `false` on disconnect for active typing targets.

## Reviews

`POST /reviews`

Protected. Creates a review after a completed booking.

Body:

```json
{
  "bookingId": "booking-id",
  "rating": 5,
  "comment": "Clear and helpful session."
}
```

Notes:

- Only booking participants can review.
- Booking must be completed.
- One review per booking per reviewer.

`GET /reviews/users/:userId`

Public. Gets user's review profile, average rating, total reviews, and review list.

## Notifications

`GET /notifications`

Protected. Lists notifications.

`GET /notifications/unread-count`

Protected. Returns unread notification count.

`PATCH /notifications/read-all`

Protected. Marks all notifications as read.

`PATCH /notifications/:notificationId/read`

Protected. Marks one notification as read.

Notification types:

- `BOOKING_CREATED`
- `BOOKING_CONFIRMED`
- `BOOKING_REJECTED`
- `BOOKING_COMPLETED`
- `NEW_MESSAGE`
- `WALLET_CREDIT`
- `WALLET_DEBIT`

## Wallet

`GET /wallet`

Protected. Gets or creates wallet summary for authenticated user.

`GET /wallet/transactions`

Protected. Lists wallet transactions.

`POST /wallet/credit`

Admin only. Credits authenticated admin's wallet in current implementation.

Body:

```json
{
  "amount": 50,
  "description": "Manual credit"
}
```

`POST /wallet/debit`

Admin only. Debits authenticated admin's wallet in current implementation.

Body:

```json
{
  "amount": 10,
  "description": "Manual debit"
}
```

## Reports

`POST /reports`

Protected. Creates a moderation report.

Body:

```json
{
  "targetType": "SKILL",
  "targetId": "uuid",
  "reason": "Spam listing",
  "details": "This looks misleading."
}
```

Target types:

- `USER`
- `SKILL`
- `REVIEW`

## Admin

All admin routes are protected and require role `Admin`.

`GET /admin/dashboard`

Returns admin counts for users, reports, and moderation state.

`GET /admin/audit/export`

Downloads moderation audit logs as CSV.

Optional query:

- `from`: date string
- `to`: date string
- `limit`: max `5000`

`GET /admin/users`

Lists users for admin.

Query:

- `search`
- `role`
- `includeDeleted`: `true` or `false`
- `page`
- `limit`: max `100`

`DELETE /admin/users/:userId`

Soft deletes a user as admin.

Body:

```json
{ "reason": "Policy violation" }
```

`GET /admin/reports`

Lists reports.

Query:

- `status`: `OPEN`, `UNDER_REVIEW`, `RESOLVED`, `DISMISSED`
- `targetType`: `USER`, `SKILL`, `REVIEW`
- `page`
- `limit`: max `100`

`PATCH /admin/reports/:reportId/status`

Updates report status.

Body:

```json
{
  "status": "RESOLVED",
  "adminNotes": "Handled by moderation."
}
```

`GET /admin/moderation/queue`

Returns flagged skills, flagged reviews, and open reports.

`POST /admin/moderation/:targetType/:targetId`

Moderates skill or review content. User moderation should use admin user routes.

Body:

```json
{
  "action": "REMOVE",
  "reason": "Spam listing",
  "reportId": "uuid"
}
```

Actions:

- `FLAG`
- `REMOVE`
- `RESTORE`

Target types:

- `SKILL`
- `REVIEW`
